# Skill：把经验封装成可复用能力

Skill 是把一套可复用做事方式封装起来，让 Agent 在合适任务中按稳定流程工作。

  <LearningFlow variant="skill" />
</ClientOnly>

## Skill 解决什么问题

复杂任务往往不只是一个 prompt：它包含触发条件、步骤、检查清单、参考资料、脚本、模板和输出规范。Skill 把这些东西打包，减少每次重新解释的成本。

## Skill 不是什么

| 不是 | 为什么 |
|---|---|
| 普通提示词 | Skill 往往包含资源、脚本、模板和多步骤流程 |
| 万能插件 | Skill 只适合稳定、可重复、边界清楚的任务 |
| 项目文档替代品 | 项目事实仍应在代码、README、spec 和规则文件中 |
| 自动正确保证 | Skill 只是提高流程稳定性，仍要验证输出 |

## Skill 里通常放什么

- 什么时候触发。
- 操作步骤和退出条件。
- 需要先读哪些文件。
- 可复用脚本、模板或资产。
- 验证方式和常见失败处理。

::: tip 什么时候值得沉淀
重复出现、流程稳定、错误代价高、团队多人复用，这四个条件满足得越多，就越值得做成 Skill。
:::

## 和 OpenSpec / aisee 的关系

需求澄清、技术上下文、change 拆分、实现、验证、复盘，都可以沉淀为 Skill。它们把“团队希望 Agent 怎么做事”变成可重复执行的流程。
