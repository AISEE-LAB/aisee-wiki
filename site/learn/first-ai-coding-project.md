# 第一个 AI Coding 项目：复刻一个网站

第一个项目不要追求复杂功能，要练会拆任务、给参考、看效果、修问题、提交成果的基本节奏。

  <LearningFlow variant="project" />
</ClientOnly>

## 项目目标

选择一个结构清晰的网站首页或落地页，学习它的信息层级和布局方式，用自拟品牌和自拟文案复刻一个 PC + 移动端可看的页面。不做登录、后台、数据库，不复制真实品牌资产。

## 模块拆解

| 模块 | AI 可以做什么 | 你要检查什么 |
|---|---|---|
| Hero | 首屏结构、标题、CTA、视觉层级 | 首屏是否清楚，移动端是否遮挡 |
| 功能介绍 | 卡片、图标、三到六个卖点 | 文案是否空泛，卡片是否过密 |
| 场景/案例 | 行业场景、案例列表、数据展示 | 数据是否虚构得合理，不冒充真实客户 |
| CTA / 定价 | 按钮、套餐卡、对比表 | 是否引导清楚，按钮是否可点 |
| FAQ | 问答结构、折叠交互 | 中文是否自然，答案是否具体 |
| Footer | 导航、联系信息、版权 | 不写不存在的备案或公司信息 |

## 推荐协作步骤

1. 让 AI 创建最小项目并说明技术栈。
2. 只做 Hero，先启动预览。
3. 用浏览器看效果，把具体问题反馈给 AI。
4. 继续做功能模块、场景模块、CTA、FAQ 和 Footer。
5. 检查移动端，不允许横向溢出和文字遮挡。
6. 跑构建或最小检查。
7. 看 diff，写清楚 Git 提交信息。

::: tip 提示词结构
目标、参考对象、技术栈、页面模块、视觉风格、验收标准、限制条件，七项都说清楚，比“帮我做个官网”稳定得多。
:::

```text
请用 Vue 3 + Vite 做一个“智能零售平台”的首页。
只学习参考站的信息层级，不复制品牌、文案和图片。
先实现 Hero 区，不做整页。
要求移动端不溢出，中文文案自然，完成后告诉我如何预览。
```

## 完成标准

- [ ] 本地能启动并看到页面。
- [ ] 首页主要模块完整。
- [ ] 移动端不崩、不横向滚动、不遮挡文字。
- [ ] 没有明显控制台报错。
- [ ] Git diff 能看懂，提交信息清楚。
- [ ] 你能说清楚 AI 做了什么、自己审查了什么。
