## 模块概述

本模块定义文档站与独立 aisee skills 仓库的集成方式，以及开源贡献、部署、SEO、LLMs.txt 和质量检查配套需求。它不要求在文档站仓库维护 skill 源码，而是要求文档站为读者提供可信的安装、版本和使用入口。

### 3.1 Skills 集成、开源部署与质量配套

<a id="fr-015"></a>

#### FR-015 提供 aisee skills 独立仓库入口与 skill 介绍页

**描述**：系统应在文档站中提供 aisee skills 独立仓库入口，并为每个核心 skill 提供介绍页，说明用途、适用边界、输入输出和与 OpenSpec 工作流的关系。

**前置条件**：FR-003 已建立 `aisee` 栏目和资源中心入口；维护者已确认 skills 源码独立仓库维护；独立仓库正式 URL 待确认。

**主流程**：
1. 读者进入 `/aisee/` 或资源中心的 skills 入口。
2. 系统展示 aisee skill 生态总览和独立 skills 仓库链接。
3. 系统列出核心 skills，并为每个 skill 提供介绍页或详情区块。
4. 读者查看 skill 的用途、触发场景、输入、输出、边界和下一步工作流。
5. 读者可从文档站跳转到独立 skills 仓库查看源码或安装说明。

**异常流程**：
- 独立 skills 仓库 URL 尚未确认 → 页面应显示待配置项，不得写死临时或错误链接。
- 某个 skill 尚未在独立仓库发布 → 页面应标注状态为规划中或待发布，不得提供不可用安装命令。
- skill 介绍与实际仓库行为不一致 → 维护者应更新版本说明或修正文档。

**验收标准**：
- [ ] `/aisee/` 栏目包含 aisee skill 生态总览。
- [ ] 文档站提供独立 skills 仓库入口，仓库 URL 可配置。
- [ ] 至少覆盖 `aisee:srs`、`aisee:ui-content`、`aisee:tech-context`、`aisee:split`、`aisee:change-design`、`aisee-init`、`aisee-opsx-schema`、`aisee-spec-migrate`、`aisee-reflect`。
- [ ] 每个 skill 说明用途、适用场景、输入、输出、禁止事项或边界。
- [ ] skill 页面说明它在 `srs -> ui-content -> tech-context -> split -> change-design` 或相关 OpenSpec 工作流中的位置。

**优先级**：P0
**依赖**：FR-003

**列表列定义**：

| 列名 | 数据来源 | 是否可排序 | 备注 |
|------|---------|-----------|------|
| Skill 名称 | skills 索引 | 是 | 如 `aisee:srs`、`aisee:split`。 |
| 所属链路 | aisee 工作流 | 是 | 需求链路、项目配置、知识复盘、资产辅助等。 |
| 用途摘要 | skill 介绍 | 否 | 一句话说明解决的问题。 |
| 发布状态 | 独立仓库版本记录 | 是 | 已发布、待发布、规划中。 |
| 仓库链接 | 独立仓库配置 | 否 | 指向 skill 源码或仓库说明。 |

**搜索与筛选条件**：

| 条件名 | 控件类型 | 匹配方式 | 默认值 |
|--------|---------|---------|--------|
| 链路类型 | 分组链接 | 精确 | 全部 |
| 发布状态 | 分组链接 | 精确 | 全部 |
| 关键词 | 输入框 | 模糊 | 无 |

**分页规格**：
- 默认每页条数：不适用，第一期 skill 数量固定且适合分组展示。
- 超大数据量时的处理：后续 skills 增多后使用分组索引和站点搜索。

**行操作**：
- 单行操作：查看 skill 介绍、查看安装命令、跳转独立仓库。
- 批量操作：无。
- 操作权限控制：公开读者可访问文档；源码访问由独立仓库权限决定。

**空状态**：
- 无数据时展示：不允许上线空 skill 生态页；至少应展示核心 skill 清单和待发布状态。
- 搜索无结果时展示：提示返回 aisee 生态总览。

#### FR-016 提供 skill 安装命令、版本说明与更新边界

**描述**：系统应为每个已发布 skill 提供安装命令、版本说明和更新边界，帮助读者理解文档描述与独立仓库版本之间的对应关系。

**前置条件**：FR-015 已建立 skill 介绍页；独立 skills 仓库的安装方式、默认分支和版本规则待维护者确认。

**主流程**：
1. 读者进入某个 skill 介绍页。
2. 系统展示该 skill 的当前文档适配版本。
3. 系统展示安装命令或安装入口。
4. 系统展示版本说明、最近更新摘要或独立仓库 changelog 链接。
5. 系统说明文档站不会内置 skill 源码，skill 更新以独立仓库为准。

**异常流程**：
- 安装命令尚未确认 → 页面应标记为待配置，不得提供无法验证的命令。
- 文档描述版本落后于独立仓库 → 页面应显示文档适配版本，并提示以仓库 release/changelog 为准。
- skill 出现破坏性更新 → 维护者应更新版本说明和相关文章中的行为描述。

**验收标准**：
- [ ] 每个已发布 skill 页面包含安装命令或安装入口。
- [ ] 每个已发布 skill 页面包含文档适配版本或版本说明。
- [ ] 页面明确说明 skill 源码在独立仓库维护。
- [ ] 页面提供版本更新来源，如 release、changelog 或仓库提交说明链接。
- [ ] 未发布 skill 不展示不可用安装命令。

**优先级**：P0
**依赖**：FR-015

**展示字段**：

| 字段名 | 展示格式 | 备注 |
|--------|---------|------|
| Skill 名称 | 文本 | 与独立仓库中的名称保持一致。 |
| 文档适配版本 | 文本 | 版本规则未确认前显示待配置。 |
| 安装命令 | 代码块 | 仅在命令确认后展示。 |
| 更新来源 | 链接 | 指向 release、changelog 或仓库说明。 |
| 更新边界 | 文本 | 说明文档站不承载源码，行为以独立仓库版本为准。 |

**可操作项**：
- 复制安装命令：读者可复制已确认命令。
- 查看版本记录：读者可跳转独立仓库版本说明。
- 查看源码：读者可跳转独立仓库对应 skill 路径。

**关联数据展示**：
- aisee 工作流：展示该 skill 所属工作流前后步骤。
- 相关文章：展示解释该 skill 方法论或使用场景的站内文章。

<a id="fr-017"></a>

#### FR-017 提供开源贡献、许可证与部署说明

**描述**：系统应提供开源贡献、许可证和部署说明，使贡献者和维护者能够理解如何修订内容、提交 PR、运行本地预览和部署到 Cloudflare Pages。

**前置条件**：FR-001 已确认 `site/` 为站点发布源，`docs/` 为协作产物目录；仓库计划公开托管在 GitHub；目标部署平台为 Cloudflare Pages。

**主流程**：
1. 贡献者进入贡献指南或仓库 README。
2. 系统说明仓库结构、内容目录、协作产物目录和禁止提交内容。
3. 系统说明本地安装、开发、预览和构建命令。
4. 系统说明文档许可证、代码示例许可证和贡献方式。
5. 维护者根据部署说明配置 GitHub Actions 或 Cloudflare Pages。

**异常流程**：
- 许可证尚未最终确认 → 文档应标记为建议方案，不得误写为已生效授权。
- 部署域名或 Cloudflare 项目尚未确认 → 部署说明保留待配置项，不写死错误项目。
- 贡献者提交私密资料或未整理草稿 → 维护者应按贡献指南拒绝或要求清理。

**验收标准**：
- [ ] README 或贡献指南说明 `site/`、`docs/`、独立 skills 仓库之间的职责边界。
- [ ] 文档包含本地开发、预览和构建命令。
- [ ] 文档包含 Cloudflare Pages 部署说明或待配置清单。
- [ ] 文档包含许可证说明，建议文档 `CC BY-SA 4.0`、代码示例 `MIT`，最终以维护者确认为准。
- [ ] 文档包含贡献流程、内容质量标准、事实源规则和敏感信息禁止项。

**优先级**：P1
**依赖**：FR-001

**页面字段规格**：

| 字段名 | 字段类型 | 必填 | 默认值 | 校验规则 | 备注 |
|--------|---------|------|--------|---------|------|
| 仓库结构说明 | 文本 | 是 | 无 | 必须说明 `site/` 与 `docs/` 边界 | skills 仓库单独说明。 |
| 本地开发命令 | 代码块 | 是 | 无 | 命令应与 package scripts 一致 | 当前脚本迁移后需同步。 |
| 构建命令 | 代码块 | 是 | 无 | 命令应与 package scripts 一致 | 构建可由维护者自行执行。 |
| 部署平台 | 文本 | 是 | Cloudflare Pages | 不写死未确认项目 ID | 说明配置项。 |
| 许可证说明 | 文本 | 是 | 建议方案 | 最终确认前标注待确认 | 文档与代码示例可分开。 |
| 贡献流程 | 列表 | 是 | 无 | 包含 Issue、PR、编辑此页 | 说明内容审核要求。 |
| 敏感信息规则 | 列表 | 是 | 无 | 禁止提交 secrets 和私有资料 | 与主文档安全要求一致。 |

**字段联动规则**：
- 当站点源目录从 `docs/` 迁移到 `site/` 后，所有开发和构建命令说明必须同步更新。
- 当许可证最终确认后，应将建议方案改为正式许可证，并同步仓库文件。

**提交行为**：
- 提交成功：贡献者可通过文档完成本地预览和 PR 提交流程。
- 提交失败：缺少命令、许可证或敏感信息规则的贡献指南不得计为完成。
- 草稿/暂存：允许部署说明保留待配置项，但不得误导为生产配置已完成。

<a id="fr-018"></a>

#### FR-018 建立质量检查、SEO、LLMs.txt 与构建说明

**描述**：系统应提供质量检查清单和站点索引能力说明，覆盖内部链接、frontmatter、SEO、sitemap、LLMs.txt、skills 外链和构建命令，便于维护者在发布前执行检查。

**前置条件**：FR-001 已建立正式站点结构；FR-017 已提供开源贡献和部署说明；用户已确认本阶段不要求代理执行截图验收或构建。

**主流程**：
1. 维护者进入质量检查说明或发布清单。
2. 系统列出发布前建议检查项。
3. 系统说明可选构建命令和本地预览命令。
4. 系统说明内部链接、外部链接、skills 安装命令和版本说明的检查口径。
5. 系统说明 SEO、sitemap、LLMs.txt 或等价 AI 友好索引的配置要求。

**异常流程**：
- 构建未执行 → 发布清单应标记为未验证，不得写成已验证。
- skills 外链或安装命令未确认 → 发布前应标记为待配置或隐藏安装命令。
- 生产域名未确认 → sitemap/canonical/hostname 配置应保留待配置项。
- LLMs.txt 插件或功能暂未启用 → 文档应说明计划和替代索引入口，不得声称已生成。

**验收标准**：
- [ ] 发布清单包含内部链接检查、外部链接检查、frontmatter/标题检查、skills 链接检查、SEO 检查和敏感信息检查。
- [ ] 文档列出本地预览和构建命令，但不声称本阶段已执行。
- [ ] 文档说明 sitemap、hostname、canonical 或等价 SEO 配置的待配置项。
- [ ] 文档说明 LLMs.txt 的目标、启用方式或待配置状态。
- [ ] 质量检查说明明确不要求截图验收，除非后续 UI/视觉 change 单独要求。

**优先级**：P1
**依赖**：FR-001、FR-017

**列表列定义**：

| 列名 | 数据来源 | 是否可排序 | 备注 |
|------|---------|-----------|------|
| 检查项 | 发布清单 | 是 | 如内部链接、SEO、skills 外链。 |
| 检查方式 | 维护说明 | 否 | 命令、人工检查或配置核对。 |
| 是否阻断发布 | 发布规则 | 是 | 阻断、警告、可延后。 |
| 负责人 | 发布流程 | 是 | 内容作者、维护者或部署负责人。 |
| 状态 | 检查记录 | 是 | 未检查、通过、失败、待配置。 |

**搜索与筛选条件**：

| 条件名 | 控件类型 | 匹配方式 | 默认值 |
|--------|---------|---------|--------|
| 检查类型 | 分组链接 | 精确 | 全部 |
| 发布阻断级别 | 分组链接 | 精确 | 全部 |
| 状态 | 分组链接 | 精确 | 全部 |

**分页规格**：
- 默认每页条数：不适用，发布清单规模固定。
- 超大数据量时的处理：后续可拆分为内容检查、构建检查、部署检查、外链检查。

**行操作**：
- 单行操作：查看检查说明、执行对应命令、打开相关配置。
- 批量操作：无。
- 操作权限控制：公开读者可查看说明；发布状态由维护者维护。

**空状态**：
- 无数据时展示：不允许上线空发布清单。
- 搜索无结果时展示：提示返回发布清单首页。

## 4. 本模块非功能补充

- skills 文档应避免与独立仓库行为漂移；版本说明必须能表达文档适配范围。
- 开源和部署说明应可被新贡献者独立理解，不依赖维护者口头说明。
- 质量检查清单应明确“已验证”和“建议执行”的区别。
- SEO 和 LLMs.txt 配置不得写死未确认域名或声称未启用功能已生效。
