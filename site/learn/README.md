## 适合谁

这部分适合三类读者：用过 AI 写代码但输出不稳定的人；想把 Codex、Claude、Gemini、GLM 等工具放进真实项目的人；准备把 AI Coding 变成团队流程，而不是一次性聊天技巧的人。

## 延伸阅读

学完这组指南后，先读 [AI Engineering](/ai-engineering/) 建立方法论框架，理解传统软件工程和 AI 增强软件工程的差异；再进入 [AISEE](/aisee/)，从 AISEE Plugin、OpenSpec 和 Compound Engineering 的组合关系理解主线实践。
