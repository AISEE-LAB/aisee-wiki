## Why

后续中文核心正文会反复解释 OpenSpec 状态机、Compound/Harness 循环、aisee 链路和软件工程流程；如果每篇文章都用临时 Mermaid、列表或手写 HTML 表达，流程展示会难以维护，也容易在术语、阶段和视觉层级上漂移。现在 `site/` 发布源和正式 IA 已建立，需要先提供一个可复用的静态流程讲解组件，作为后续内容生产的共享表达能力。

## What Changes

- 新增一个 VuePress/Plume 可用的流程讲解组件，用于在 Markdown 内容中展示阶段、节点、连接关系、说明文本和强调状态。
- 提供适合 OpenSpec、Compound/Harness、aisee 工作机制和通用软件工程流程的最小数据结构或组件使用接口。
- 提供至少一个中文使用示例或示例页，方便后续 P0/P1 中文正文复用。
- 保证组件可在静态站点构建环境中运行，不依赖后端、数据库、运行时 API 或在线保存能力。
- 保证组件在桌面和移动端可读；移动端可降级为纵向步骤、分组列表或简化连接关系。
- 保证组件只承担“展示和解释流程”的职责，不扩展为拖拽编辑器、图形编辑器、流程持久化系统或所有文章的最终流程图内容。

## Capabilities

### New Capabilities

- `reusable-flow-explainer`: 定义静态文档站中可复用流程讲解组件的可观察行为，包括 Markdown 使用方式、流程阶段/节点/连接/状态表达、示例内容、响应式降级和范围边界。

### Modified Capabilities

- 无。

## Impact

- 影响 `site/.vuepress/` 下的主题组件、客户端注册、样式或类型定义。
- 可能新增 `site/workflows/` 或现有栏目下的组件示例页面，也可能更新相关栏目入口以链接示例。
- 可能影响后续中文核心内容 change 的写作方式，但本 change 不编写 P0/P1 正文。
- 不引入新的后端服务、数据库、鉴权、运行时 API、在线编辑能力或第三方图形库承诺。
