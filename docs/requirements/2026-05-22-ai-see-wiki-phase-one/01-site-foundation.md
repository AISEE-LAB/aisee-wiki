## 模块概述

本模块定义 AI SEE Wiki 第一期站点基础结构、首页入口、栏目导航、公开内容边界和全站阅读路径。它为中文正文、英文双语体系、skills 集成和开源部署配套提供稳定的信息架构基础。

### 3.1 站点基础与信息架构

<a id="fr-001"></a>

#### FR-001 建立正式站点源目录与栏目结构

**描述**：系统应提供可用于第一期上线的正式文档站目录结构，覆盖中文主站、英文站、核心栏目、资源中心和后续内容扩展位置。

**前置条件**：项目已存在 VuePress/Plume 基础工程，维护者已确认第一期栏目范围包含 `/learn/`、`/openspec/`、`/compound/`、`/aisee/`、`/workflows/`、`/thinking/`、`/resources/` 和 `/en/`。

**主流程**：
1. 维护者将站点发布源迁移到 `site/`。
2. 系统在 `site/` 下创建中文栏目首页、英文栏目首页和资源入口目录。
3. 系统移除或隔离默认示例内容，避免示例页面出现在正式导航中。
4. 系统保留后续文章、资源、模板、案例和英文内容的可扩展路径。

**异常流程**：
- 源目录迁移未完成 → 系统不得把 `docs/` 内协作产物作为站点发布源，并应阻断正式发布准备。
- 栏目目录缺失或命名不一致 → 系统不得将对应栏目加入正式导航，并提示维护者补齐。
- 默认示例内容仍被导航引用 → 系统应视为上线阻断问题，直到示例入口被替换或移除。

**验收标准**：
- [ ] 站点存在中文主站入口和英文 `/en/` 入口。
- [ ] 中文栏目至少覆盖 `learn`、`openspec`、`compound`、`aisee`、`workflows`、`thinking`、`resources`。
- [ ] 英文目录使用与中文对应的 slug。
- [ ] 默认 demo 页面不作为正式首屏、导航或学习路径入口。
- [ ] 站点发布源为 `site/`，协作产物保留在独立 `docs/`。
- [ ] README、贡献说明或部署说明中明确 `site/` 与 `docs/` 的职责边界。

**优先级**：P0
**依赖**：无

<a id="fr-002"></a>

#### FR-002 配置中文首页与核心学习路径入口

**描述**：系统应提供中文首页，清晰表达 AI SEE Wiki 定位，并引导读者进入 OpenSpec、Compound/Harness、aisee skill 生态和推荐学习路径。

**前置条件**：FR-001 已建立正式站点目录；站点名称、中文定位和核心栏目范围已确认。

**主流程**：
1. 读者访问中文根路径 `/`。
2. 系统展示站点名称、定位、副标题和核心主张。
3. 系统提供 OpenSpec、Compound & Harness、aisee Skill Ecosystem 三个核心方向入口。
4. 系统提供从 AI Coding 基础到综合案例的推荐学习路径。
5. 读者可从首页进入对应栏目首页或首批核心文章。

**异常流程**：
- 首批核心文章尚未完成 → 首页入口应指向栏目首页或明确的建设中说明，不得产生 404。
- 某个核心方向暂缺内容 → 系统应保留入口并说明当前可阅读内容，不得伪造已完成文章。
- 首页文案与站点定位不一致 → 维护者应以主文档范围和规划文档定位为准修正文案。

**验收标准**：
- [ ] 首页 H1 为 `AI SEE Wiki` 或经维护者确认的站点品牌名。
- [ ] 首页明确表达“AI Engineering 的开放文档站”和“OpenSpec × Compound Engineering × aisee skills”的关系。
- [ ] 首页提供三个核心方向入口：OpenSpec、Compound & Harness、aisee Skill Ecosystem。
- [ ] 首页提供推荐学习路径，至少覆盖基础、OpenSpec、Compound/Harness、aisee、综合案例。
- [ ] 首页所有内部链接可解析到存在页面或明确的占位说明页。

**优先级**：P0
**依赖**：FR-001

**展示字段**：

| 字段名 | 展示格式 | 备注 |
|--------|---------|------|
| 站点名 | 文本 | `AI SEE Wiki`，如品牌后续调整需同步全站。 |
| 中文定位 | 文本 | `AI Engineering 的开放文档站`。 |
| 核心主张 | 文本 | 表达本站不是工具教程站，而是 AI 时代软件工程方法论站。 |
| 三个核心方向 | 链接列表 | OpenSpec Deep Dive、Compound & Harness、aisee Skill Ecosystem。 |
| 推荐学习路径 | 有序列表 | 从基础能力到综合案例。 |
| 开源入口 | 链接 | 指向 GitHub 仓库、贡献说明或编辑入口。 |

**可操作项**：
- 进入栏目：读者点击核心方向或学习路径项后进入对应栏目首页。
- 查看开源入口：读者点击开源链接后进入仓库、贡献说明或编辑页面。

<a id="fr-003"></a>

#### FR-003 配置栏目导航、集合与资源中心入口

**描述**：系统应配置正式导航、文档集合和资源中心入口，使读者能按栏目浏览核心文档、观点文章、模板、命令速查和示例资源。

**前置条件**：FR-001 已建立栏目结构；中文首页的主要学习入口已定义。

**主流程**：
1. 维护者配置中文导航，包含核心文档栏目和资源入口。
2. 维护者配置英文导航，保持与中文导航结构一致。
3. 系统将 `learn`、`openspec`、`compound`、`aisee`、`workflows`、`resources` 作为文档型内容组织。
4. 系统将 `thinking` 作为观点或文章型内容组织。
5. 读者可通过导航、栏目首页、资源中心进入首批内容。

**异常流程**：
- 导航引用不存在页面 → 系统应在质量检查中识别为内部链接问题。
- 栏目集合类型与内容用途冲突 → 维护者应按“文档型内容”和“观点型内容”边界调整。
- 资源中心条目尚未完成 → 页面应说明资源状态和后续补齐路径，不得链接到不存在文件。

**验收标准**：
- [ ] 中文导航覆盖规划中的核心栏目和资源中心。
- [ ] 英文导航结构与中文导航保持一致，允许正文按阶段补齐。
- [ ] 栏目首页说明该栏目定位、内容范围和下一步阅读路径。
- [ ] 资源中心至少提供模板、Schema Pack、命令速查、检查清单、示例仓库、术语表、参考资料等入口。
- [ ] 导航不得引用默认 demo 页面作为正式内容。

**优先级**：P0
**依赖**：FR-001

**列表列定义**：

| 列名 | 数据来源 | 是否可排序 | 备注 |
|------|---------|-----------|------|
| 栏目名称 | 站点信息架构 | 否 | 如 Learn、OpenSpec、Compound、aisee。 |
| 栏目路径 | 站点路由 | 否 | 中文根路径与英文 `/en/` 下应使用对应 slug。 |
| 栏目角色 | 规划文档 | 否 | 标识基础层、核心主线、综合实践、观点或资源。 |
| 内容状态 | 内容维护记录 | 否 | 标识已完成、建设中、待补充。 |

**搜索与筛选条件**：

| 条件名 | 控件类型 | 匹配方式 | 默认值 |
|--------|---------|---------|--------|
| 栏目类型 | 链接分组 | 精确 | 全部栏目 |
| 内容语言 | 语言路径 | 精确 | 中文 |

**分页规格**：
- 默认每页条数：不适用，栏目导航为固定规模。
- 超大数据量时的处理：不适用。

**行操作**：
- 单行操作：进入栏目首页。
- 批量操作：无。
- 操作权限控制：无登录权限要求，公开读者可访问。

**空状态**：
- 无内容时展示：栏目说明和建设中提示。
- 搜索无结果时展示：站点本地搜索返回无结果提示。

<a id="fr-004"></a>

#### FR-004 建立公开内容与协作产物的隔离规则

**描述**：系统应明确公开站点内容、协作产物、skills 源码和本地参考资料的边界，避免未整理草稿或私有资料被误发布。

**前置条件**：FR-001 已确认站点发布源迁移到 `site/`；维护者已确认 skills 源码独立仓库维护。

**主流程**：
1. 维护者定义 `site/` 为站点公开发布源目录。
2. 维护者定义 `docs/` 为需求文档、UI 内容规格、技术上下文、复盘和学习沉淀等协作产物目录。
3. 系统在 README 或贡献说明中说明哪些目录参与站点构建，哪些目录不直接发布。
4. 系统要求从协作产物迁移到公开站点前必须人工整理、脱敏和改写。
5. 系统记录 skills 源码不随文档站发布，文档站只引用独立仓库公开信息。

**异常流程**：
- 协作产物目录与站点源目录冲突 → 系统应按 `site/` 发布、`docs/` 协作的已确认约束调整目录。
- 页面引用本地绝对路径作为读者可访问链接 → 系统应要求改为公开仓库链接、相对站内链接或参考说明。
- 文档中包含未脱敏私有内容 → 系统不得发布该页面，直到维护者完成清理。

**验收标准**：
- [ ] `site/` 作为公开站点源目录、`docs/` 作为协作产物目录的职责在文档中明确。
- [ ] `aisee-*` skills 源码不进入文档站仓库交付范围。
- [ ] 公开页面不得要求读者访问维护者本地 Obsidian 路径。
- [ ] 需求文档、草稿和复盘资料若位于仓库内，应默认不作为站点导航入口。
- [ ] 贡献说明包含内容公开前的脱敏和事实核查要求。

**优先级**：P1
**依赖**：FR-001

**角色定义**：

| 角色名 | 描述 | 数据可见范围 |
|--------|------|------------|
| 公开读者 | 访问已发布站点内容的人。 | 仅公开站点页面和公开资源。 |
| 文档贡献者 | 通过 Issue/PR 提交公开内容修订的人。 | 公开仓库中允许编辑的站点内容。 |
| 站点维护者 | 负责整理草稿、合并 PR、发布站点的人。 | 公开内容、协作产物、部署配置和来源资料索引。 |

**功能权限矩阵**：

| 功能点 | 公开读者 | 文档贡献者 | 站点维护者 |
|--------|---------|------------|------------|
| 查看公开站点内容 | 是 | 是 | 是 |
| 查看未发布协作产物 | 否 | 按仓库权限 | 是 |
| 提交公开内容修订 | 否 | 是 | 是 |
| 将协作产物整理为公开页面 | 否 | 可提议 | 是 |
| 修改发布配置 | 否 | 否 | 是 |

**无权限时的处理**：
- 功能入口：公开站点不展示私有协作产物入口。
- 直接访问受限内容：由仓库权限或托管平台权限控制，公开站点不提供绕过入口。

<a id="fr-005"></a>

#### FR-005 提供站点级术语表与阅读路径

**描述**：系统应提供站点级术语表和跨栏目阅读路径，帮助读者理解 OpenSpec、Compound Engineering、Harness、aisee skill 等核心概念，并顺畅进入后续文章。

**前置条件**：FR-002 已定义首页学习路径；FR-003 已建立栏目导航和资源中心入口。

**主流程**：
1. 读者从首页、资源中心或文章内链接进入术语表。
2. 系统展示核心术语、中文解释、英文推荐译法和关联阅读。
3. 系统在栏目首页和核心文章末尾提供下一步阅读路径。
4. 读者可根据基础、OpenSpec、Compound/Harness、aisee、workflows 的顺序继续阅读。
5. 维护者在新增文章时更新相关术语和阅读路径。

**异常流程**：
- 同一术语在不同文章中解释冲突 → 以站点级术语表为准，并修正冲突页面。
- 英文译法暂未确认 → 系统在术语表标记为待确认，不应在正文中多种译法混用。
- 关联阅读文章尚未完成 → 系统指向栏目首页或已完成的前置文章。

**验收标准**：
- [ ] 术语表包含 OpenSpec、Compound Engineering、Harness、aisee skill、Spec-first、Delta Spec、Artifact、Knowledge Compounding 等核心术语。
- [ ] 术语表中的 OpenSpec、Compound Engineering、Harness 使用主文档确认后的定义。
- [ ] 核心栏目首页包含下一步阅读建议。
- [ ] P0/P1 文章末尾包含与主题相关的下一步阅读链接。
- [ ] 中英文术语映射可被英文内容模块复用。

**优先级**：P1
**依赖**：FR-002、FR-003

**列表列定义**：

| 列名 | 数据来源 | 是否可排序 | 备注 |
|------|---------|-----------|------|
| 术语 | 术语表 | 是 | 中文正文中使用的术语或英文原词。 |
| 定义 | 术语表 | 否 | 简洁解释，不替代专题文章。 |
| 英文推荐译法 | 双语术语表 | 是 | 英文正文保持一致使用。 |
| 关联栏目 | 信息架构 | 是 | 指向 OpenSpec、Compound、aisee、workflows 等栏目。 |
| 推荐阅读 | 文章索引 | 否 | 指向已完成或规划中的文章。 |

**搜索与筛选条件**：

| 条件名 | 控件类型 | 匹配方式 | 默认值 |
|--------|---------|---------|--------|
| 关键词 | 输入框 | 模糊 | 无 |
| 关联栏目 | 链接分组 | 精确 | 全部 |

**分页规格**：
- 默认每页条数：不适用，术语表按主题分组展示。
- 超大数据量时的处理：当术语数量显著增长时，优先通过站点搜索和锚点分组支持查找。

**行操作**：
- 单行操作：进入关联阅读或专题文章。
- 批量操作：无。
- 操作权限控制：公开读者可访问，维护者通过仓库流程修改。

**空状态**：
- 无数据时展示：不允许上线空术语表；至少应包含核心术语。
- 搜索无结果时展示：提示读者返回资源中心或使用站点全文搜索。

## 4. 本模块非功能补充

- 站点信息架构应优先稳定 URL，避免第一期上线后频繁移动核心栏目路径。
- 首页、栏目首页和资源入口应使用语义化 Markdown 标题层级，便于搜索、无障碍访问和 LLM 引用。
- 导航和栏目配置应易于维护者扩展，不应要求每新增文章都修改多个无关配置文件。
- 公开内容与协作产物的隔离规则应优先降低误发布风险。
