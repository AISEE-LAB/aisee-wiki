## ADDED Requirements

### Requirement: 替换默认首页示例
系统 MUST 用 AI SEE Wiki 自定义首页替换 Plume 初始化的默认首页示例。

#### Scenario: 中文首页不展示 Plume 示例内容
- **WHEN** 读者访问中文首页 `/`
- **THEN** 系统不得展示 `Theme Plume`、`VuePress Next Theme`、示例博客按钮或 Plume GitHub 示例入口作为正式首页内容

#### Scenario: 英文首页不展示 Plume 示例内容
- **WHEN** 读者访问英文首页 `/en/`
- **THEN** 系统不得展示 Plume 初始化示例 hero、示例博客按钮或 Plume GitHub 示例入口作为正式首页内容

### Requirement: 中文首页表达核心价值主张
系统 MUST 在中文首页表达 AI SEE Wiki 的核心价值主张：AI 时代的软件工程方法论。

#### Scenario: 中文首页首屏传达方法论定位
- **WHEN** 读者访问中文首页首屏
- **THEN** 系统必须展示“AI 时代的软件工程方法论”或等价的中文主张

#### Scenario: 中文首页说明方法论与工具关系
- **WHEN** 读者查看中文首页核心说明
- **THEN** 系统必须表达工具和技能需要由方法论驾驭，而不是只展示工具教程入口

#### Scenario: 中文首页正确描述 aisee
- **WHEN** 读者查看中文首页对 aisee 的描述
- **THEN** 系统必须把 aisee 表达为本站探索并沉淀的工作机制，不得把它误写为与 OpenSpec 或 Compound Engineering 同等成熟的外部标准

### Requirement: 首页提供长页面内容槽位
系统 MUST 把首页组织为可扩展的长页面，而不是把所有信息堆叠在首屏。

#### Scenario: 首页包含工程领域区块
- **WHEN** 读者浏览中文首页
- **THEN** 系统必须提供 AI 增强工程领域的内容区块，覆盖 Web、App、Desktop、嵌入式、硬件编程和复杂业务系统等方向或等价表达

#### Scenario: 首页包含方法论判断区块
- **WHEN** 读者浏览中文首页
- **THEN** 系统必须提供“工具越强，越需要方法论”或等价判断，并覆盖需求契约、设计边界、验证 gate、知识沉淀等核心概念

#### Scenario: 首页包含三条核心主线
- **WHEN** 读者浏览中文首页
- **THEN** 系统必须提供 OpenSpec、Compound Engineering、aisee 工作机制三个核心主线入口或说明

#### Scenario: 首页包含推荐学习路径
- **WHEN** 读者浏览中文首页
- **THEN** 系统必须提供从基础能力到 Spec-first、OpenSpec、Harness、验证闭环、知识复利和综合实践的推荐学习路径或等价路径

#### Scenario: 首页包含开放协作入口
- **WHEN** 读者浏览中文首页
- **THEN** 系统必须提供开放协作或开源参与入口，包括文档修订、真实案例、workflow / skill / schema 模板、术语与参考资料等方向或等价表达

### Requirement: 首页正式入口链接有效
系统 MUST 在首页提供指向正式栏目和资源入口的有效链接。

#### Scenario: 中文首页链接正式栏目
- **WHEN** 维护者检查中文首页链接
- **THEN** 首页必须链接到存在的正式栏目或资源入口，不得链接到 demo/blog 示例页面作为正式入口

#### Scenario: 英文首页链接英文正式栏目
- **WHEN** 维护者检查英文首页链接
- **THEN** 英文首页必须链接到 `/en/` 下存在的正式栏目或资源入口，不得链接到英文 demo/blog 示例页面作为正式入口

### Requirement: 英文首页保持对应结构
系统 MUST 为 `/en/` 提供与中文首页对应的英文首页结构。

#### Scenario: 英文首页有对应入口
- **WHEN** 英文读者访问 `/en/`
- **THEN** 系统必须展示 AI SEE Wiki 的英文首页入口，而不是空页或中文首页

#### Scenario: 英文首页允许内容待完善
- **WHEN** 英文首页完整正文尚未完成
- **THEN** 系统必须提供英文结构、基础说明或待完善状态，不得伪装为英文完整正文已完成

### Requirement: 首页视觉资产使用边界清晰
系统 MUST 使用可维护的前端结构承载首页内容，并把视觉素材限定为装饰层。

#### Scenario: 首页不使用整页切图承载内容
- **WHEN** 维护者检查首页实现
- **THEN** 首页正文、标题、按钮、卡片和链接必须由代码或 Markdown 承载，不得作为整页图片或含文字切图承载

#### Scenario: 首页不直接依赖协作目录资源
- **WHEN** 维护者检查站点发布源
- **THEN** 首页运行时不得直接依赖 `docs/design-assets/` 路径下的协作产物；如需使用素材，必须复制或迁移到公开站点发布源

#### Scenario: 首页装饰素材不替代真实内容
- **WHEN** 首页使用 SVG、PNG 或其他视觉素材
- **THEN** 素材只能作为装饰层或辅助视觉，真实文字、链接和结构必须保持可访问、可搜索和可维护

### Requirement: 首页响应式可读
系统 MUST 在桌面、平板和移动端保持首页内容可读、入口可用。

#### Scenario: 移动端不强行展示复杂视觉图
- **WHEN** 读者在移动端访问首页
- **THEN** 系统必须优先保证文字、入口和学习路径可读，可隐藏或简化复杂方法论地图等装饰视觉

#### Scenario: 首页内容不发生严重重叠
- **WHEN** 读者在常见桌面、平板或移动端宽度访问首页
- **THEN** 首页主要文字、按钮、卡片和链接不得出现严重重叠或不可点击状态
