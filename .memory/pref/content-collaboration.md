# 文章协作偏好

**日期：** 2026-05-27
**类型：** pref

## 摘要

对教程页、学习路径页和其他文章内容的完善或重写，维护者希望先看“计划稿 / 内容规划”，确认方向后再进入正式编写。目标和计划一旦确认，后续执行必须严格围绕已确认目标推进，不再偏离到新的表达方式或额外方向。

## 详情

- 当任务是“完善文章”“优化正文”“重写某篇内容”“补图并改文案”这类内容创作型工作时，默认先给出计划稿，而不是直接动正文。
- 计划稿应至少说明：
  - 文章的核心结论
  - 预计的章节结构
  - 是否需要新增组件、图示或截图
  - 哪些内容会删减、收束或强调
- 只有在维护者确认计划方向后，才进入实际编写。
- 进入实际编写后，默认严格按已确认的目标和计划执行；不要自行切换表达方式、补做未确认内容，或等维护者重复提醒“按原目标做”。
- 这条偏好优先适用于 `site/learn/` 下的学习路径文章，也可推广到其他教程型栏目。
