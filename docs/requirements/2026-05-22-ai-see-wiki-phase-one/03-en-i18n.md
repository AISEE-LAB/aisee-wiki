## 模块概述

本模块定义 AI SEE Wiki 英文内容与双语体系的需求。英文站不是中文站的机械镜像，而是面向英文读者重写核心方法论，同时保持栏目结构、slug、术语和职责边界与中文主线一致。

### 3.1 英文内容与双语体系

<a id="fr-011"></a>

#### FR-011 建立英文站点首页与栏目入口

**描述**：系统应提供英文首页和英文栏目入口，使英文读者能理解 AI SEE Wiki 的定位、核心主题和推荐学习路径。

**前置条件**：FR-001 已建立 `site/` 发布源和英文 `/en/` 目录；FR-002 已完成中文首页定位；英文 locale 已在站点配置中启用或计划启用。

**主流程**：
1. 英文读者访问 `/en/`。
2. 系统展示英文站点标题、英文定位和核心主张。
3. 系统展示 OpenSpec、Compound & Harness、aisee Skill Ecosystem 三个英文核心入口。
4. 系统展示英文推荐学习路径。
5. 读者可进入 `/en/learn/`、`/en/openspec/`、`/en/compound/`、`/en/aisee/`、`/en/workflows/`、`/en/thinking/`、`/en/resources/`。

**异常流程**：
- 某个英文栏目正文尚未完成 → 系统应提供栏目入口和状态说明，不得产生 404。
- 英文首页与中文首页定位不一致 → 维护者应以主文档范围和中文首页核心主张为准调整。
- 英文页面误用中文读者上下文 → 内容作者应改写背景说明，使英文读者能独立理解。

**验收标准**：
- [ ] `/en/` 首页存在并可访问。
- [ ] 英文首页表达 `An open documentation site for AI Engineering` 或维护者确认的等价定位。
- [ ] 英文首页明确 OpenSpec、Compound Engineering、aisee skills、Spec-first、Harness Engineering、knowledge compounding 的关系。
- [ ] 英文首页包含与中文一致的核心栏目入口。
- [ ] 英文首页不只是中文首页逐字翻译，应补足英文读者所需上下文。

**优先级**：P1
**依赖**：FR-001、FR-002

**展示字段**：

| 字段名 | 展示格式 | 备注 |
|--------|---------|------|
| Site name | Text | `AI SEE Wiki`。 |
| Positioning | Text | `An open documentation site for AI Engineering`。 |
| Core themes | Link list | OpenSpec、Compound & Harness、aisee Skill Ecosystem。 |
| Learning path | Ordered list | Mirrors Chinese structure while using English explanations. |
| Open source entry | Link | Points to public repository, contribution guide, or edit link. |

**可操作项**：
- Enter section：读者点击核心入口后进入对应英文栏目。
- Switch language：读者可从英文页面进入对应中文页面或中文栏目。

<a id="fr-012"></a>

#### FR-012 完成英文 P0/P1 正文

**描述**：系统应交付 P0/P1 英文正文，使英文站最终具备与中文站一致的核心内容覆盖范围。

**前置条件**：FR-006 和 FR-007 已完成中文 P0/P1 正文；FR-011 已建立英文首页和栏目入口；FR-013 已定义术语映射规则。

**主流程**：
1. 内容作者读取已完成中文 P0/P1 正文。
2. 内容作者根据英文读者语境重写英文正文。
3. 系统保持英文文章主题、slug 和结论与中文对应文章一致。
4. 系统在英文文章中补充英文读者需要的背景、术语解释和案例上下文。
5. 系统将英文正文发布到 `/en/` 下对应栏目路径。

**异常流程**：
- 中文正文尚未完成 → 英文正文不得独立凭空扩展，应等待中文事实源稳定或标记待补。
- 英文正文只是机器直译且上下文不足 → 不得计为完成，应重写为英文读者可读版本。
- 英文正文与中文正文结论冲突 → 维护者应确认是否中文正文需要更新，或英文正文需要修正。
- 英文文章引用中文本地语境过重 → 内容作者应补充解释或替换为通用英文读者可理解的表达。

**验收标准**：
- [ ] 完成 P0 六篇文章的英文正文。
- [ ] 完成 P1 六篇文章的英文正文。
- [ ] 英文正文与中文正文共享主题、结论、职责边界和核心术语。
- [ ] 英文正文不是逐字翻译，应针对英文读者重写背景和解释。
- [ ] 每篇英文文章包含下一步阅读链接和对应中文页面链接。

**优先级**：P1
**依赖**：FR-006、FR-007、FR-011

**列表列定义**：

| 列名 | 数据来源 | 是否可排序 | 备注 |
|------|---------|-----------|------|
| English title | 英文文章索引 | 是 | 与中文主题对应，可采用英文读者更自然的标题。 |
| Chinese source | 中文文章索引 | 是 | 指向对应中文正文。 |
| Slug | 路由规则 | 是 | 与中文同主题页面保持一致。 |
| Content status | 内容维护记录 | 是 | Not started、Draft、Review、Published。 |
| Terminology check | 双语术语表 | 是 | 标识术语是否已按 FR-013 检查。 |

**搜索与筛选条件**：

| 条件名 | 控件类型 | 匹配方式 | 默认值 |
|--------|---------|---------|--------|
| Priority | Link group | Exact | P0/P1 |
| Section | Link group | Exact | All |
| Status | Link group | Exact | All |

**分页规格**：
- 默认每页条数：不适用，P0/P1 英文文章数量固定为 12 篇。
- 超大数据量时的处理：后续英文内容增多后，使用栏目索引、标签和站点搜索。

**行操作**：
- 单行操作：Open English page、Open Chinese source、Open edit link。
- 批量操作：无。
- 操作权限控制：公开读者可阅读；内容修改通过仓库 PR 或维护者流程完成。

**空状态**：
- 无数据时展示：英文栏目应展示建设状态和中文原文链接。
- 搜索无结果时展示：提示返回英文栏目首页或中文原文。

<a id="fr-013"></a>

#### FR-013 建立中英文术语映射与 slug 同步规则

**描述**：系统应提供中英文术语映射和 slug 同步规则，保证中英文内容在核心概念、路径和交叉引用上保持一致。

**前置条件**：FR-005 已提供站点级术语表；FR-011 已建立英文栏目入口。

**主流程**：
1. 维护者从中文术语表提取核心术语。
2. 系统为每个核心术语记录英文推荐写法、中文定义和英文解释。
3. 系统要求同主题中英文页面使用一致 slug。
4. 内容作者在英文正文中按术语表使用固定译法。
5. 系统在中英文页面之间提供对应关系。

**异常流程**：
- 某个术语存在多个可选译法 → 维护者应在术语表中选定主译法，并记录必要说明。
- 英文页面 slug 与中文页面不一致 → 维护者应修正路径或记录重定向策略。
- 英文正文出现未登记的新核心术语 → 内容作者应补充术语表后再发布。

**验收标准**：
- [ ] 术语表包含 OpenSpec、Compound Engineering、Harness、Spec-first、Delta Spec、Artifact、aisee skill、knowledge compounding 等核心术语。
- [ ] 中英文同主题页面使用相同 slug，例如 `/openspec/delta-spec/` 与 `/en/openspec/delta-spec/`。
- [ ] 英文正文统一使用 `knowledge compounding` 表达“知识复利”。
- [ ] aisee 语境中的“技能”统一使用 `skill`，不随意替换为 `capability`。
- [ ] 术语表可从中文资源中心和英文资源中心访问。

**优先级**：P1
**依赖**：FR-005、FR-011

**列表列定义**：

| 列名 | 数据来源 | 是否可排序 | 备注 |
|------|---------|-----------|------|
| 中文术语 | 术语表 | 是 | 中文站使用的术语。 |
| English term | 术语表 | 是 | 英文站固定写法。 |
| 中文定义 | 术语表 | 否 | 以主文档和中文正文为准。 |
| English explanation | 术语表 | 否 | 面向英文读者的解释。 |
| Related slug | 路由索引 | 是 | 指向对应专题页或栏目。 |

**搜索与筛选条件**：

| 条件名 | 控件类型 | 匹配方式 | 默认值 |
|--------|---------|---------|--------|
| Keyword | Input | Fuzzy | None |
| Related section | Link group | Exact | All |
| Translation status | Link group | Exact | All |

**分页规格**：
- 默认每页条数：不适用，术语表按主题分组展示。
- 超大数据量时的处理：后续可增加字母索引或站点搜索辅助。

**行操作**：
- 单行操作：Open related article、Open Chinese definition、Open English explanation。
- 批量操作：无。
- 操作权限控制：公开读者可访问；维护者通过仓库流程修改。

**空状态**：
- 无数据时展示：不允许上线空术语表；至少应包含核心术语。
- 搜索无结果时展示：提示返回资源中心。

<a id="fr-014"></a>

#### FR-014 配置双语 SEO、语言切换与缺页处理策略

**描述**：系统应支持中英文页面的基础 SEO、语言切换和英文缺页处理，保证双语站点可被读者、搜索引擎和 AI 工具稳定理解。

**前置条件**：FR-011 已建立英文首页和栏目入口；FR-013 已定义 slug 同步规则；站点部署域名和仓库信息待维护者确认。

**主流程**：
1. 维护者配置站点标题、描述、域名和基础 SEO 信息。
2. 系统为中英文页面维护对应关系。
3. 系统提供语言切换入口或等价的跨语言链接。
4. 当英文正文尚未完成时，系统展示英文建设中说明并链接中文原文。
5. 系统为核心页面提供 LLM/SEO 友好的标题、描述和结构化内容。

**异常流程**：
- 生产域名尚未确认 → SEO 配置使用待配置项，不阻塞内容编写，但不得写死错误域名。
- 英文对应页缺失 → 系统展示英文准备中说明，并链接中文原文或对应栏目。
- 中英文页面主题不一致 → 维护者应修正页面映射或移除语言切换链接。

**验收标准**：
- [ ] 中英文首页和核心栏目具备语言、标题和描述配置。
- [ ] 中英文同主题页面可互相发现。
- [ ] 英文缺页时展示 `English version is being prepared` 或维护者确认的等价说明，并链接中文原文。
- [ ] SEO 配置不得写死未确认的生产域名。
- [ ] 页面结构便于生成 sitemap、LLMs.txt 或等价索引。

**优先级**：P1
**依赖**：FR-011、FR-013

**页面字段规格**：

| 字段名 | 字段类型 | 必填 | 默认值 | 校验规则 | 备注 |
|--------|---------|------|--------|---------|------|
| Page title | 文本 | 是 | 无 | 不为空，能表达页面主题 | 中英文分别配置。 |
| Description | 文本 | 是 | 无 | 不为空，避免与正文无关 | 用于 SEO 和页面摘要。 |
| Locale | 枚举 | 是 | 中文 `/` | `/` 或 `/en/` | 与 VuePress locale 一致。 |
| Canonical URL | 文本 | 否 | 无 | 域名确认后配置 | 域名未确认前不得写死错误值。 |
| Alternate language URL | 文本 | 否 | 无 | 对应页存在时配置 | 缺页时指向中文原文或栏目。 |
| English placeholder message | 文本 | 否 | English version is being prepared | 仅英文缺页使用 | 必须包含中文原文链接。 |

**字段联动规则**：
- 当英文正文未完成时，英文页面应显示缺页说明，并链接中文原文或对应中文栏目。
- 当生产域名确认后，应补齐 canonical、sitemap 和语言替代链接配置。

**提交行为**：
- 提交成功：页面元信息进入站点配置或页面 frontmatter。
- 提交失败：缺少 title、description 或 locale 的页面不得计为双语 SEO 完成。
- 草稿/暂存：允许英文草稿存在，但不得冒充已完成正文。

## 4. 本模块非功能补充

- 英文正文应面向英文读者重写，不应只做逐字翻译。
- 英文内容应保持术语一致，避免同一概念在不同文章中出现多个译法。
- 中英文 URL 应尽量稳定，避免后续迁移破坏已发布链接。
- 英文缺页说明应清楚、克制，不应制造已完成的误导。
