## 模块概述

本模块定义 AI SEE Wiki 第一期中文内容交付范围和质量标准。中文内容是第一阶段的内容生产主线，需完成 P0/P1 正文，并为英文翻译、资源中心、后续 OpenSpec change 拆分提供稳定事实源。

#### FR-006 完成中文 P0 核心文章正文

**描述**：系统应交付规划文档中 P0 级别的中文核心文章正文，使站点上线时能直接体现 OpenSpec 深度、Compound/Harness 和 aisee 生态三条主线。

**前置条件**：FR-001 已完成正式站点目录结构；FR-003 已完成栏目导航和集合入口；维护者允许读取本地参考目录作为内容事实源。

**主流程**：
1. 内容作者读取规划文档中 P0 文章清单。
2. 内容作者按文章主题读取本地 Obsidian 参考目录中的相关资料。
3. 内容作者为每篇 P0 文章生成可发布正文，而不是占位页或大纲。
4. 系统将正文放入对应中文栏目路径。
5. 系统在栏目首页和文章末尾加入下一步阅读入口。

**异常流程**：
- 某篇文章缺少足够参考资料 → 文章不得虚构具体工具行为，应标注待补事实点或降级为待确认问题。
- 文章内容偏离本站定位 → 维护者应按“方法论优先、工具只是入口”的原则重写。
- 文章正文只包含大纲或占位文案 → 该文章不得计入 P0 完成。

**验收标准**：
- [ ] 完成《OpenSpec 是什么：AI 编程时代为什么需要规范层》中文正文。
- [ ] 完成《OpenSpec 基础工作流：从 propose 到 archive》中文正文。
- [ ] 完成《Spec 粒度设计：项目级、模块级、能力级如何划分》中文正文。
- [ ] 完成《Compound Engineering Plugin 是什么》中文正文。
- [ ] 完成《OpenSpec × Compound 的职责边界》中文正文。
- [ ] 完成《aisee skill 生态总览》中文正文。
- [ ] 每篇文章包含明确问题、核心结论、适用边界和下一步阅读。

**优先级**：P0
**依赖**：FR-001、FR-003

**列表列定义**：

| 列名 | 数据来源 | 是否可排序 | 备注 |

| 文章标题 | P0 内容清单 | 是 | 使用规划文档确认的标题，允许实现阶段微调但不得改变主题。 |

- 默认每页条数：不适用，P0 文章数量固定为 6 篇。
- 超大数据量时的处理：不适用。

**行操作**：
- 单行操作：进入文章页、进入编辑入口、查看关联参考说明。
- 批量操作：无。

**空状态**：

#### FR-008 建立中文文章写作标准和事实源标注规则

**描述**：系统应提供中文文章写作标准和事实源标注规则，确保不同作者产出的正文在结构、语气、术语、边界和可追溯性上保持一致。

**前置条件**：FR-006 已开始中文正文生产；维护者已确认内容应优先基于规划文档和本地 Obsidian 参考目录改写。

**主流程**：
1. 维护者定义中文文章统一结构。
2. 内容作者按统一结构撰写 P0/P1 正文。
3. 内容作者记录文章使用的主要事实源。
4. 系统要求涉及 OpenSpec、Compound、aisee 的文章明确其在整体工程系统中的职责位置。
5. 系统要求文章末尾提供下一步阅读路径。

**异常流程**：
- 文章缺少适用边界 → 内容作者应补充“不适用场景”或“容易误用的场景”。
- 文章混淆规范源、工具输出和个人判断 → 内容作者应拆分表述并明确来源类型。
- 文章直接暴露本地绝对路径 → 发布前应改写为公开引用说明或维护者内部参考记录。

**验收标准**：
- [ ] 每篇中文 P0/P1 文章至少包含“这篇解决什么问题”“核心结论”“背景与动机”“工作机制”“常见误区”“适用边界”“下一步阅读”。
- [ ] 涉及流程的文章应使用 Mermaid 或等价文本说明流程关系。
- [ ] 涉及 OpenSpec / Compound / aisee 的文章必须说明该主题在系统中的职责边界。
- [ ] 文章不得写成单纯命令搬运或 prompt 清单。
- [ ] 文章事实源记录不得要求公开读者访问维护者本地路径。

**优先级**：P0
**依赖**：FR-006

**页面字段规格**：

| 字段名 | 字段类型 | 必填 | 默认值 | 校验规则 | 备注 |
|--------|---------|------|--------|---------|------|
| 标题 | 文本 | 是 | 无 | 应准确表达文章主题 | 可与规划标题微调但不得偏离主题。 |
| 解决的问题 | 文本 | 是 | 无 | 至少 1 段 | 说明读者为什么需要读这篇。 |
| 核心结论 | 文本 | 是 | 无 | 至少 1 条 | 避免空泛判断。 |
| 背景与动机 | 文本 | 是 | 无 | 至少 1 段 | 说明问题来源和上下文。 |
| 工作机制 | 文本 | 是 | 无 | 需要覆盖关键概念关系 | 可配 Mermaid。 |
| 常见误区 | 列表 | 是 | 无 | 至少 1 条 | 体现边界和反例。 |
| 适用边界 | 列表 | 是 | 无 | 至少 1 条 | 说明什么时候不适合。 |
| 与三条主线关系 | 文本 | 是 | 无 | 必须提及相关主线 | 根据文章主题覆盖 OpenSpec、Compound、aisee。 |
| 下一步阅读 | 链接列表 | 是 | 无 | 至少 1 个站内链接 | 指向已完成或栏目首页。 |
| 事实源记录 | 文本 | 是 | 无 | 不暴露本地绝对路径给公开读者 | 可在维护文档中保留内部来源。 |

**字段联动规则**：
- 当文章涉及流程、DAG、工作流或职责边界时，应提供 Mermaid 图或结构化流程说明。
- 当文章引用未公开本地资料时，公开正文只能表达来源类型或改写后的事实，不展示本地绝对路径。

**提交行为**：
- 提交成功：文章进入对应栏目，并更新栏目首页或阅读路径。
- 提交失败：维护者根据缺失字段、术语冲突或事实源问题退回修改。
- 草稿/暂存：支持在协作产物目录或草稿分支维护，但草稿不得进入正式导航。

#### FR-009 建立中文资源页、模板页和命令速查入口

**描述**：系统应提供中文资源中心入口，集中展示模板、Schema Pack、命令速查、检查清单、示例仓库、术语表和参考资料，支撑读者从阅读转向实践。

**前置条件**：FR-003 已配置资源中心入口；FR-006 已完成至少一批核心正文用于链接资源页。

**主流程**：
1. 读者进入 `/resources/` 中文资源中心。
2. 系统展示资源分类和对应入口。
3. 读者可进入模板、Schema Pack、命令速查、检查清单、示例仓库、术语表、参考资料页面。
4. 系统在核心文章中链接相关资源页。
5. 维护者可按资源成熟度逐步补充内容。

**异常流程**：
- 资源尚未整理完成 → 页面应标注建设中和计划内容，不得提供不可用下载或无效链接。
- 资源来自协作草稿 → 公开前必须人工整理、脱敏和改写。
- 资源与文章结论冲突 → 维护者应优先修正文档或资源说明中的职责边界。

**验收标准**：
- [ ] `/resources/` 中文入口存在并可从导航访问。
- [ ] 资源中心包含模板库、Schema Pack、命令速查、检查清单、示例仓库、术语表、参考资料入口。
- [ ] 至少术语表和命令速查入口可被 P0/P1 中文文章引用。
- [ ] 资源页明确哪些内容已可用、哪些内容待补齐。
- [ ] 资源页不得直接公开未整理的协作产物。

**优先级**：P1
**依赖**：FR-003、FR-006

**列表列定义**：

| 列名 | 数据来源 | 是否可排序 | 备注 |
|------|---------|-----------|------|
| 资源名称 | 资源中心索引 | 是 | 如模板库、命令速查、术语表。 |
| 资源类型 | 资源分类 | 是 | 模板、Schema、命令、清单、示例、术语、参考。 |
| 可用状态 | 资源维护记录 | 是 | 已可用、建设中、待整理。 |
| 关联文章 | 文章索引 | 否 | 指向引用该资源的 P0/P1 文章。 |
| 来源说明 | 维护记录 | 否 | 标注来自公开资料、项目整理或本地参考改写。 |

**搜索与筛选条件**：

| 条件名 | 控件类型 | 匹配方式 | 默认值 |
|--------|---------|---------|--------|
| 资源类型 | 分组链接 | 精确 | 全部 |
| 可用状态 | 分组链接 | 精确 | 全部 |

**分页规格**：
- 默认每页条数：不适用，第一期资源分类固定。
- 超大数据量时的处理：当资源数量增加后，使用分类页和站点搜索支持查找。

**行操作**：
- 单行操作：查看资源详情、进入关联文章。
- 批量操作：无。
- 操作权限控制：公开读者可访问已发布资源；维护者通过仓库流程修改。

**空状态**：
- 无数据时展示：不允许上线空资源中心；至少应展示资源分类和建设状态。
- 搜索无结果时展示：提示返回资源中心首页。

#### FR-010 为中文正文建立文章间阅读路径

**描述**：系统应在中文首页、栏目首页和 P0/P1 文章之间建立阅读路径，使读者能按“基础能力 → OpenSpec → Compound/Harness → aisee → workflows → resources”的顺序继续学习。

**前置条件**：FR-006 和 FR-007 已完成中文 P0/P1 正文；FR-005 已提供站点级术语表和阅读路径基础。

**主流程**：
1. 读者从首页进入推荐学习路径。
2. 系统将读者导向对应栏目首页或第一篇前置文章。
3. 读者阅读文章后，在文末看到下一步阅读建议。
4. 系统在栏目首页展示该栏目内推荐顺序和跨栏目下一站。
5. 读者可通过资源中心补充模板、命令、术语和示例材料。

**异常流程**：
- 下一篇文章尚未完成 → 系统应链接到栏目首页或标注建设中，不得产生 404。
- 文章之间推荐顺序与内容依赖冲突 → 维护者应根据 P0/P1 依赖关系调整。
- 资源页未完成 → 文末可指向资源中心入口并标注待补内容。

**验收标准**：
- [ ] 中文首页包含全站推荐学习路径。
- [ ] 每个核心栏目首页包含栏目内阅读顺序。
- [ ] 每篇 P0/P1 中文文章末尾包含至少 1 个下一步阅读链接。
- [ ] 关键跨栏目链路覆盖 OpenSpec、Compound/Harness、aisee 和 workflows。
- [ ] 阅读路径不得指向不存在页面。

**优先级**：P1
**依赖**：FR-006、FR-007

**展示字段**：

| 字段名 | 展示格式 | 备注 |
|--------|---------|------|
| 当前文章 | 文本或链接 | 标识读者当前所在主题。 |
| 前置阅读 | 链接列表 | 可为空，但复杂主题建议提供。 |
| 下一步阅读 | 链接列表 | 每篇 P0/P1 文章至少 1 个。 |
| 关联资源 | 链接列表 | 指向模板、术语、命令或示例。 |
| 所属学习阶段 | 文本 | 基础、OpenSpec、Compound/Harness、aisee、workflows、resources。 |

**可操作项**：
- 进入下一篇：读者点击文末链接进入推荐文章。
- 返回栏目首页：读者可从文章页回到栏目路径。
- 查看资源：读者可进入关联资源页补充实践材料。

**关联数据展示**：
- 术语表：展示当前文章关键术语的站内链接。
- 资源中心：展示与当前主题相关的模板、命令或检查清单入口。

## 4. 本模块非功能补充

- 中文正文应优先追求观点密度、实践细节和可操作边界，不做空泛工具介绍。
- 每篇文章应避免未经确认的具体实现承诺；涉及工具行为、命令、schema 或 workflow 时应基于参考资料。
- 公开正文不得暴露维护者本地绝对路径、私人笔记路径或未授权内容。
- 内容结构应便于后续英文重写和术语同步。
