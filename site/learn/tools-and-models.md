# AI 编程工具与模型怎么选

不要问“哪个工具最好”，而要问“当前任务需要哪种工具形态和模型能力”。本文事实口径核对时间：2026-05-25；安装、价格、额度和平台支持以官方文档为准。

<ToolLogoGrid />

## 先按工具形态分类

| 形态 | 代表 | 适合任务 | 选择重点 |
|---|---|---|---|
| 编辑器补全 | GitHub Copilot | 日常续写、函数补全、小改动 | 低打扰、补全质量、编辑器集成 |
| AI-first 编辑器 | Cursor、Windsurf | 页面开发、存量代码问答、局部重构 | 上下文索引、diff 体验、团队迁移成本 |
| Agent CLI/App | Codex、Claude Code、Gemini CLI | 多步骤任务、跑命令、修构建、读仓库 | 权限控制、命令执行、diff、长任务稳定性 |
| Web Chat | ChatGPT、Claude、Gemini、GLM | 方案讨论、解释概念、生成草稿 | 便捷度、上下文长度、多模态、资料来源 |

## 再看模型能力

| 能力维度 | 为什么重要 |
|---|---|
| 代码生成与修改 | 决定基础产出质量，但不能替代项目上下文 |
| 长上下文 | 决定能否理解大型仓库、长文档和多文件变更 |
| 工具调用 | 决定模型能否通过 Agent 执行真实动作 |
| 多模态 | UI、截图、设计稿、报错截图会明显受益 |
| 长任务稳定性 | 多轮执行时能否不跑偏、不遗忘约束 |
| 中文表达与本地化 | 国内团队、中文需求和中文文档会更顺手 |

## 本站为什么重点讲这四类

| 推荐对象 | 核心定位 | 适合谁 |
|---|---|---|
| Codex | 以项目、线程、diff、命令和权限为中心的 AI Coding Agent | 想以真实仓库和工程流程为主线的读者 |
| Claude Code | 终端原生 Agent，工程协作与长任务体验成熟 | 已习惯 CLI、重视代码库理解和团队规则的人 |
| Gemini | 长上下文、多模态、搜索和 Google 生态结合 | 需要读大项目、图片/文档参与或 Google 生态的人 |
| GLM | 中文友好、成本敏感、多工具兼容生态 | 中文团队、预算敏感或希望补充国产模型路线的人 |

::: tip 官方资料口径
- Codex 官方说明强调桌面 App、CLI、IDE 和云端入口，以及默认权限边界和 diff 审查。
- Claude Code 官方文档提供原生安装、npm 安装、更新和签名校验说明。
- Gemini CLI 官方仓库提供 npm、Homebrew、npx 等入口，以及 Google 账号、API Key、Vertex AI 等认证方式。
- Z.AI 官方文档把 GLM Coding Plan 定位为 AI-powered coding 订阅包，并列出支持工具和 MCP 能力。
:::

## 选择矩阵

| 场景 | 优先看什么 | 建议 |
|---|---|---|
| AI Coding 新手 | 操作入口、权限提示、diff 可读性 | 从 Codex 主线开始，先练安全任务 |
| 真实项目维护 | 读仓库、跑命令、改文件、验证 | Codex 或 Claude Code，必须配 Git 和项目规则 |
| UI / 设计稿实现 | 多模态、浏览器预览、截图反馈 | Gemini、Codex + 浏览器工具、或支持设计上下文的 Agent |
| 中文团队 | 中文表达、成本、账号与网络便利性 | Codex 主线 + GLM/Gemini/Claude 作为补充 |
| 团队流程 | 项目规则、审查、权限、可复用流程 | 重点看 Agent 的 Memory、Skill、Hook 和自动化能力 |

这不是固定排名。随着模型和产品快速变化，选型应该围绕任务，而不是围绕品牌。
