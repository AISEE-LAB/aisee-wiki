# OpenSpec Spec 粒度设计图逻辑

## 表达目标

说明主 specs 的粒度不是固定目录层级，而是按行为影响范围判断：全局约束、领域行为、独立能力分别适合不同边界。

## 读者视角

读者已经理解 OpenSpec artifact 写法，接下来需要知道主 specs 应该如何划分边界，并把这件事和 change 中的 delta specs 区分开。

## 图类型

使用 `FlowExplainer` timeline 模式。图形表达三种判断尺度：项目级 -> 模块级 -> 能力级。

## 节点

- 项目级：全局约束，影响多个领域。
- 模块级：领域行为，围绕稳定维护边界。
- 能力级：具体能力，可以长期独立维护和理解。

## 连线

- 项目级 -> 模块级：全局规则约束领域行为。
- 模块级 -> 能力级：领域边界决定能力归属。

## 不展示内容

- 不把三种粒度画成强制目录层级。
- 不把主 spec 粒度和 delta spec/change 拆分混成一件事。
- 不展开上游规划链路。
- 不展示具体 CLI 命令。

## 组件选择

使用已有 `FlowExplainer`。该组件能在移动端保持可读，并用文本结构承载图示内容。
