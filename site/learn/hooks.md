# Hook：在关键节点加上自动检查和约束

Hook 是 Agent 工作流中的自动化检查点，用来在关键时刻注入上下文、拦截风险、运行验证或记录结果。

  <LearningFlow variant="hooks" />
</ClientOnly>

## 为什么需要 Hook

Agent 执行速度快，但也容易越界、漏验证或忘记项目规则。Hook 把一些“每次都应该做”的动作放进生命周期，而不是靠人反复提醒。

## 常见位置

| 位置 | 能做什么 |
|---|---|
| 任务开始前 | 注入项目规则、检查 change 状态、加载上下文 |
| 工具调用前 | 拦截危险命令、检查权限、确认路径 |
| 文件修改后 | 检查敏感信息、格式、命名、范围 |
| 验证前后 | 自动运行测试、记录失败原因 |
| 提交前 | 检查工作区、提交信息、无关文件 |
| 归档前 | 确认任务、验证、基线文档都完成 |

## Hook 能做什么

- 注入规则：提醒 Agent 读取 `AGENTS.md`、OpenSpec 工件和 memory 索引。
- 拦截风险：阻止删除、重置、上传密钥、生产操作。
- 运行验证：格式化、构建、测试、链接检查。
- 记录结果：把复盘、失败原因、可复用经验写到合适位置。

::: warning Hook 的边界
Hook 不应该把所有流程都变成硬阻断，也不能替代人的判断。它适合处理明确、重复、高风险或容易遗忘的检查。
:::

## 与项目规则的关系

`AGENTS.md`、`CLAUDE.md`、OpenSpec 状态机和 Hook 共同形成约束层。规则告诉 Agent 应该怎么做，Hook 在关键节点帮助它不忘记、不越界、不漏验。
